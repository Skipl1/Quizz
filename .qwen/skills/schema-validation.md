# Schema-Based Validation Skill

## Инструкции
1. **Zod/Joi Setup:** Используй библиотеку Zod для описания входящих данных.
2. **Input Sanitization:** Всегда проверяй текстовые поля (имена игроков, тексты вопросов) на XSS.
3. **Type Safety:** Генерируй TypeScript типы на основе схем для использования и на фронтенде, и на бэкенде.
4. **Error Handling:** Если данные не валидны, отправляй клиенту стандартизированную ошибку `invalid-payload`.

## Примеры
- "Создай схему валидации для события `submit-answer`, проверяя, что ID вопроса — число, а ответ не пуст."
- "Защити создание квиза: имя не должно быть длиннее 50 символов и содержать HTML."
